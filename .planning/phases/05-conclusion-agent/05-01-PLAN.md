# Plan 05-01: Binary Verdict Fields and Conflict-Aware Rule

**Goal:** Extend `Verdict` with `verdict_binary` (`REAL`|`FAKE`) and `verdict_label_vi` (`Thật`|`Giả`), then update `conclusion_agent.py` to compute them from the evidence graph and model results.

## must_haves

- `state.py` `Verdict` adds `verdict_binary` and `verdict_label_vi` optional fields.
- `conclusion_agent.py` reads `state["evidence_graph"]` and detects cross-source conflict (trusted vs flagged/social).
- Binary rule: SUPPORTED/REAL/TRUE → REAL; REFUTED/FAKE/FALSE/MISLEADING/UNVERIFIED/NEI → FAKE.
- Cross-source conflict overrides to FAKE/`Giả`.
- Both LLM and fallback paths return the new fields.
- Tests cover mapping, conflict override, and fallback with no models.

## Task 05-01-T2: Implement conflict detection and binary mapping helpers

<task>
<id>05-01-T2</id>
<wave>1</wave>
<type>execute</type>
<action>
In `factcheck_agents/agents/conclusion_agent.py` add:

```python
_BINARY_REAL_LABELS = {"SUPPORTED", "REAL", "TRUE"}
_BINARY_FAKE_LABELS = {"REFUTED", "FAKE", "FALSE", "MISLEADING", "UNVERIFIED", "NEI"}


def _has_cross_source_conflict(evidence_graph: Optional[Any]) -> bool:
    if evidence_graph is None:
        return False
    tiers = {
        data.get("source_tier")
        for _, data in evidence_graph.graph.nodes(data=True)
        if data.get("node_type") == "evidence"
    }
    return "trusted" in tiers and bool(tiers & {"flagged", "social"})


def _map_to_binary(label: str, conflict: bool) -> Tuple[str, str]:
    if conflict:
        return "FAKE", "Giả"
    label_upper = str(label).upper()
    if label_upper in _BINARY_REAL_LABELS:
        return "REAL", "Thật"
    return "FAKE", "Giả"
```
</action>
