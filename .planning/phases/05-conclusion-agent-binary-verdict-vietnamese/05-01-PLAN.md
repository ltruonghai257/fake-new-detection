# Plan 05-01: state.py Verdict Fields + conclusion_agent.py Binary Rule and Evidence Graph

**Goal:** Add `verdict_binary` and `verdict_label_vi` to the `Verdict` TypedDict in `state.py`, then extend `conclusion_agent.py` with: a `_BINARY_MAP` constant + `_apply_binary_verdict()` helper, tier-annotated `_format_evidence()`, Vietnamese `_fallback_verdict()`, and graph conflict detection + binary mapping in `conclusion_agent()`.

## must_haves

- `Verdict` TypedDict in `state.py` has `verdict_binary: Literal["REAL", "FAKE"]` and `verdict_label_vi: Literal["Thật", "Giả"]` (both `total=False`, no existing fields changed)
- `_BINARY_MAP = {"TRUE": "REAL", "REAL": "REAL", "SUPPORTED": "REAL"}` defined at module level in `conclusion_agent.py`
- `_apply_binary_verdict("TRUE") == ("REAL", "Thật")`, `_apply_binary_verdict("NEI") == ("FAKE", "Giả")`, `_apply_binary_verdict("MISLEADING") == ("FAKE", "Giả")`, `_apply_binary_verdict("UNVERIFIED") == ("FAKE", "Giả")`
- `_format_evidence()` prepends `[TRUSTED]`/`[FLAGGED]`/`[SOCIAL]`/`[UNKNOWN]` to each evidence line
- `_fallback_verdict()` returns `verdict_binary` and `verdict_label_vi` on both code paths; `rationale` and `recommendation` strings are in Vietnamese
- `conclusion_agent()` iterates `evidence_graph.graph.edges(data=True)` in a `try/except` block and sets `graph_override_fake=True` if a `contradicts` edge with `source_tier="trusted"` targeting the `"statement"` node is found (per D-01, D-02)
- If `graph_override_fake`, `verdict_binary="FAKE"` and `verdict_label_vi="Giả"` override the LLM-derived mapping
- `uv run python -c "from factcheck_agents.agents.conclusion_agent import conclusion_agent"` exits 0
- `uv run pytest tests/factcheck_agents/ -q --ignore=tests/processing/coolant` exits 0 (no regressions)

**Update `_format_evidence(evidence)`** (in-place):

For each evidence item `e`, read `e.get("source_tier", "")` and look it up in `_TIER_LABELS` (fallback `"[UNKNOWN]"`). Prepend the label to each formatted line. Example output line: `[1] [TRUSTED] VnExpress article — https://vnexpress.net/...\n    snippet text`.

**Update `_fallback_verdict(model_results, evidence)`** (in-place, both return paths):

No-model path — add Vietnamese strings and binary fields:
- `rationale`: `"Không có mô hình nào khả dụng và không có LLM để đánh giá bằng chứng."`
- `recommendation`: `"Cần cấu hình checkpoint mô hình và/hoặc khóa LLM, hoặc xem xét bằng chứng thủ công."`
- Call `_apply_binary_verdict("UNVERIFIED")` to get `verdict_binary`, `verdict_label_vi` and add both to the returned `Verdict`.

Has-model path — add Vietnamese strings and binary fields:
- `rationale`: `f"Kết quả phỏng đoán dựa trên {top['model']} ({top.get('label')})."`
- `recommendation`: `"Kết quả heuristic; hãy bật LLM để có suy luận được cân nhắc từ bằng chứng."`
- The `label_map` result (mapped label) is passed to `_apply_binary_verdict()` to get `verdict_binary`, `verdict_label_vi`; add both to the returned `Verdict`.
- Keep existing `label_map = {"REFUTED": "FALSE", "FAKE": "FALSE", "SUPPORTED": "TRUE", "REAL": "TRUE", "NEI": "UNVERIFIED"}` — only the `label` field uses it.

**Update `conclusion_agent(state)`** (in-place):

After extracting `evidence`, add `evidence_graph = state.get("evidence_graph")`.

Add graph conflict detection block BEFORE the `llm = get_llm()` call (per D-01, D-02):
```
graph_override_fake = False
if evidence_graph is not None:
    try:
        for src, dst, edge_data in evidence_graph.graph.edges(data=True):
            if (
                edge_data.get("type") == "contradicts"
                and dst == "statement"
                and evidence_graph.graph.nodes[src].get("source_tier") == "trusted"
            ):
                graph_override_fake = True
                break
    except Exception:
        pass
```

In the `llm is None` branch: after building the fallback verdict, if `graph_override_fake` is True, override `verdict["verdict_binary"] = "FAKE"` and `verdict["verdict_label_vi"] = "Giả"`.

In the LLM exception branch: same override pattern after calling `_fallback_verdict()`.

In the normal LLM path (after parsing `data`): extract `llm_label = str(data.get("label", "UNVERIFIED")).upper()`, call `verdict_binary, verdict_label_vi = _apply_binary_verdict(llm_label)`, then apply graph override if `graph_override_fake`. Add `verdict_binary=verdict_binary` and `verdict_label_vi=verdict_label_vi` to the `Verdict(...)` constructor call.

Update the messages log line in the LLM success branch to: `f"[Conclusion] {verdict['label']} ({verdict['confidence']:.2f}) → {verdict_binary}"`.
</action>

<threat_model>
No new network calls, no new external dependencies, no new API endpoints. Threat surface is limited to internal Python transformations on already-fetched state.

- **T-05-01 (Information Disclosure): LLM-returned `rationale`** — LLM output is stored verbatim in `Verdict.rationale`. Rationale is user-visible output from the user's own configured LLM; no cross-user data exposure. Disposition: accept.
- **T-05-02 (Tampering): `source_tier` on graph nodes trusted by conflict detection** — `source_tier="trusted"` is set by `search_agent` based on `classify_domain()` using env-configured `FACTCHECK_TRUSTED_DOMAINS`. An attacker who controls the search results could spoof a trusted-tier node, triggering a `graph_override_fake`. Disposition: accept — the graph is per-request in-memory; no persistence or privilege boundary crossed.
- **T-05-03 (Denial of Service): Malformed `evidence_graph`** — `conclusion_agent` wraps graph traversal in `try/except Exception: pass`, so any graph corruption falls back gracefully. Disposition: mitigated.
- **T-05-SC (Tampering): Supply chain** — no new packages installed in this plan. Disposition: accept.
</threat_model>
